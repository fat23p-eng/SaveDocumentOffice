# ระบบจัดเก็บไฟล์เอกสารสำนักงาน

เว็บแอป + Google Sheet สำหรับจัดเก็บ ค้นหา และจัดการไฟล์เอกสารของสำนักงานสหกรณ์
ไฟล์จริงเก็บอยู่บน Google Drive ส่วน Google Sheet ทำหน้าที่เป็นฐานข้อมูล (metadata)
และเป็นที่อยู่ของ Google Apps Script ที่รันเป็นเว็บแอป

## โครงสร้างไฟล์ในโปรเจกต์นี้

| ไฟล์ | หน้าที่ |
|---|---|
| `ระบบจัดเก็บไฟล์เอกสารสำนักงาน.xlsx` | Template ฐานข้อมูล อัพโหลดเข้า Drive แล้วเปิดด้วย Google Sheets เพื่อแปลงเป็น Google Sheet |
| `Code.gs` | โค้ดฝั่งเซิร์ฟเวอร์ (Google Apps Script) — ล็อกอิน, อัพโหลดไฟล์ขึ้น Drive, ค้นหา/กรอง, แก้ไข, ลบ |
| `Index.html` | หน้าเว็บแอป (login + อัพโหลดแบบลากวาง + ค้นหา/กรองเอกสาร) ธีมกรมท่า/ส้ม/ทอง/ขาว |

## โครงสร้างชีตในไฟล์ Google Sheet

- **เริ่มต้นใช้งาน** — คำแนะนำการติดตั้งภาษาไทย
- **ข้อมูลหลัก** — ฐานข้อมูลเอกสาร: วันที่อัพโหลด, กลุ่มที่อัพโหลด, ประเภท, บันทึกช่วยจำ, Files (ลิงก์), ชื่อไฟล์
- **users** — รายชื่อกลุ่ม/รหัสผ่าน/สิทธิ์ (คอลัมน์: ชื่อกลุ่ม, รหัสผ่าน, สิทธิ์)
- **ประเภทเอกสาร** — รายการประเภทเอกสารสำหรับ dropdown (แก้ไข/เพิ่มเองได้)

## ระบบสิทธิ์ (Role-based access)

กำหนดในคอลัมน์ "สิทธิ์" ของชีต `users` มี 3 ระดับ:

| สิทธิ์ | ความสามารถ |
|---|---|
| `ทั่วไป` (ค่าเริ่มต้นถ้าไม่กรอก) | เห็น/ลบได้เฉพาะเอกสารของกลุ่มตนเอง **บวกกับ** กลุ่มที่กำหนดให้ทุกคนเห็นได้เสมอ (ดูหัวข้อด้านล่าง) |
| `ดูทุกกลุ่ม` | เห็นเอกสารของทุกกลุ่มได้ แต่ลบ/แก้ไขได้เฉพาะของกลุ่มตนเอง |
| `ผู้ดูแลระบบ` | เห็น / แก้ไข / ลบเอกสารของทุกกลุ่มได้ทั้งหมด และอัพโหลดแทนกลุ่มอื่นได้ |

การตรวจสิทธิ์ทำที่ฝั่งเซิร์ฟเวอร์ (`Code.gs`) ทั้งหมด ไม่ใช่แค่ซ่อนปุ่มฝั่งหน้าเว็บ
ดังนั้นแก้ JS ในเบราว์เซอร์ก็ไม่สามารถมองเห็น/ลบข้ามกลุ่มที่ไม่มีสิทธิ์ได้

### กลุ่มที่ทุกคนเห็นได้เสมอ

กำหนดไว้ในโค้ดที่ตัวแปรนี้ในไฟล์ `Code.gs`:

```js
const ALWAYS_VISIBLE_GROUPS = ['งานนายทะเบียนสหกรณ์', 'งานสำนักงาน'];
```

ชื่อกลุ่มในตัวแปรนี้ต้องตรงกับชื่อในชีต `users` เป๊ะ ทั้งตัวสะกดและการเว้นวรรค
ถ้าต้องการเปลี่ยนกลุ่ม หรือเพิ่ม/ลดจำนวนกลุ่มที่เห็นได้เสมอ แก้ที่ตัวแปรนี้ได้เลย

## วิธี Deploy

1. อัพโหลดไฟล์ xlsx เข้า Google Drive แล้วเปิดด้วย **Google Sheets** (คลิกขวา > เปิดด้วย > Google Sheets) เพื่อแปลงเป็น Google Sheet
2. ในชีตที่แปลงแล้ว ไปที่เมนู **Extensions > Apps Script**
3. ลบโค้ดเดิมใน `Code.gs` ของโปรเจกต์ทั้งหมด แล้ววางเนื้อหาจาก `Code.gs` ในโปรเจกต์นี้แทน
4. สร้างไฟล์ HTML ใหม่ในโปรเจกต์ ชื่อไฟล์ต้องเป็น **`Index`** พอดี (ไม่ใส่ `.html`) แล้ววางเนื้อหาจาก `Index.html`
5. กด **Deploy > New deployment** เลือกประเภทเป็น **Web app**
   - Execute as: **Me**
   - Who has access: **Anyone** (หรือ Anyone within [องค์กร] ถ้าใช้ Google Workspace)
6. กด Deploy แล้วคัดลอกลิงก์เว็บแอปไปใช้งาน

> ทุกครั้งที่แก้โค้ดใน Apps Script editor ต้องไปที่ **Deploy > Manage deployments > (ไอคอนดินสอ) > New version > Deploy**
> ไม่อย่างนั้นเว็บแอปที่ deploy ไว้แล้วจะยังรันโค้ดเวอร์ชันเก่าอยู่

## การย้ายไปติดตั้งที่ Google Account อื่น

แนะนำ 2 วิธี:

**วิธีที่ 1 — โอนกรรมสิทธิ์ไฟล์ (เก็บข้อมูลเดิมไว้ทั้งหมด)**
แชร์ทั้งไฟล์ Google Sheet และโฟลเดอร์ Drive ที่เก็บเอกสารให้ account ใหม่ในสิทธิ์ Editor ก่อน
แล้วเปลี่ยนสิทธิ์เป็น "โอนความเป็นเจ้าของ" (Make owner) จากนั้น deploy เว็บแอปใหม่จาก account ปลายทาง

**วิธีที่ 2 — ทำสำเนาใหม่ (Make a copy)**
ล็อกอินด้วย account ใหม่ เปิดไฟล์ที่แชร์มา แล้วใช้ **File > Make a copy**
สคริปต์ที่ผูกไว้ (bound script) จะติดไปด้วยอัตโนมัติ โฟลเดอร์ Drive สำหรับเก็บไฟล์เอกสารจะถูกสร้างขึ้นใหม่
อัตโนมัติใน Drive ของ account ใหม่ตั้งแต่การอัพโหลดไฟล์แรก (เพราะ Script Properties ไม่ติดไปกับการ Make a copy)
เหมาะสำหรับกรณีใช้ไฟล์นี้เป็น Template แล้วต้องการเริ่มฐานข้อมูลใหม่สะอาดๆ ในอีก account

ไม่ว่าจะวิธีไหน ต้อง **Deploy ใหม่เสมอ** หลังเปลี่ยนเจ้าของไฟล์/สคริปต์ เพราะลิงก์เว็บแอปและสิทธิ์การรัน
("Execute as: Me") จะผูกกับคนที่กด deploy ล่าสุดเท่านั้น

## หมายเหตุด้านความปลอดภัย

ไฟล์ที่อัพโหลดจะตั้งสิทธิ์แชร์เป็น "ทุกคนที่มีลิงก์ดูได้" (`DriveApp.Access.ANYONE_WITH_LINK`)
ถ้าต้องการจำกัดเฉพาะคนในองค์กร (กรณีใช้ Google Workspace) แก้ค่านี้ในฟังก์ชัน `uploadFile` ของ `Code.gs`
เป็น `DriveApp.Access.DOMAIN_WITH_LINK`
